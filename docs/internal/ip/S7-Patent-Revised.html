1. Field of Invention

This invention relates to artificial intelligence inference systems, and more particularly to the Convergence Weight Schema (CWS) — a unified architecture derived from a single geometric first principle — the dual-curve convergence geometry — whose behavior at any point on a continuous axis deterministically governs ternary quantization, token reliability, model consensus, semantic memory organization, and retrieval verification across a complete sovereign offline AI platform. Unlike prior art systems that apply statistical approximation independently to each AI subsystem, the CWS derives every component from one geometric foundation, producing a system whose behavior is deterministic, auditable, and geometrically verifiable at each inference step.

This invention integrates five novel methods into a single coherent system: (1) geometric ternary weight assignment for large language model quantization; (2) bidirectional token discernment for hallucination detection; (3) multi-model witness consensus with RLHF bandage detection; (4) constant-time semantic plane memory replacing quadratic attention; and (5) convergence vector decomposition enabling direction-verified retrieval-augmented reasoning. The five methods are not independent — they share a common geometric foundation and are designed to operate as an integrated pipeline.

2. Background

2.1 The Five Problems This Invention Solves

Problem 1 — Quantization without geometry: Large language models contain billions of floating-point parameters. Existing ternary quantization methods assign weights based on statistical distribution (absmean thresholding), treating each weight independently without geometric context. This discards inter-parameter relationships and provides no mechanism for verifying assignment quality.

Problem 2 — Unverified token generation: Language models generate tokens autoregressively without built-in mechanisms to verify whether output is reliable. Post-hoc fact-checking is slow and requires external knowledge bases. No existing method evaluates token reliability bidirectionally through the same geometric space.

Problem 3 — Single-model trust: No single language model can be fully trusted. RLHF training further complicates reliability by teaching models to produce human-pleasing responses ("bandaging") that may contradict the model's genuine base knowledge. Existing verification uses single-model self-consistency or simple majority voting among identical copies.

Problem 4 — Quadratic attention cost: Transformer-based language models use self-attention that scales as O(n²) with sequence length. This limits context windows, requires expensive hardware, and provides no semantic organization of retrieved information.

Problem 5 — Scalar retrieval without verification: Retrieval-augmented generation (RAG) uses cosine similarity — a scalar metric — to retrieve memory entries. Scalar similarity answers "how close?" but not "in which direction?" No retrieval system verifies that a retrieved memory is geometrically appropriate for the current reasoning step.

2.2 The Common Root

All five problems share a common root: AI systems operate on statistical approximations without a geometric foundation that makes their decisions deterministic, verifiable, and auditable. This invention provides that foundation.

4. Detailed Description

4.1 The Dual Convergence Curves

The geometric foundation of the entire system is two complementary mathematical curves:

Structure Curve (Father):

S(x) = exp(-|x - (-1)|) * cos(0.8x) Decay center: -1.0. Peaks at x = -1 (the structure pole, foundation).

Nurture Curve (Mother):

N(x) = exp(-|x - (+1)|) * cos(0.8x) Decay center: +1.0. Peaks at x = +1 (the nurture pole, growth).

For any value x on the convergence axis [-∞, +∞]:

structure_val = S(x) nurture_val = N(x) convergence_w = N(x) - S(x) [signed dominance weight] door_distance = |x| [distance from the convergence point x=0]

The Door (x = 0): The point where both curves intersect and neither dominates. Maximum uncertainty. The neutral gate state. All convergence passes through the Door.

Ternary Assignment from Curve Values:

  • IF structure_val dominates AND convergence_w < threshold_low: weight = -1
  • IF nurture_val dominates AND convergence_w > threshold_high: weight = +1
  • IF neither dominates (near x = 0, the Door): weight = 0

This single rule governs weight assignment (Component 1), token discernment (Component 2), retrieval direction (Component 5), and all intermediate geometric computations.

4.3 Component 2: ForToken/RevToken Bidirectional Discernment

4.3.1 Forward Pass (ForToken)

For each generated token at index i:

forward_value = evaluate_convergence(token[i], context[0..i-1])

Evaluates the token's convergence position given all preceding context.

4.3.2 Reverse Pass (RevToken)

reverse_value = evaluate_convergence(token[i], context[i+1..n])

Evaluates the same token given all following context, independent of the forward pass.

Temporal Architecture Note: The RevToken reverse evaluation is a post-generation verification pass applied to a completed token sequence. It does not require real-time access to future tokens during generation; instead, it is applied as a verification layer over the completed output before any FERTILE/BABEL classification is recorded and before any result is stored or returned to the caller. This two-pass architecture is an intentional design feature: generation proceeds forward, verification proceeds bidirectionally. The system therefore operates as a generate-then-verify pipeline, not a real-time streaming classifier. The INSERT-only storage covenant ensures that no FERTILE classification is recorded until the RevToken pass is complete.

4.3.3 Discernment Computation

agreement = 1.0 - |forward_value - reverse_value| / max(|forward_value|, |reverse_value|) IF agreement >= FERTILE_THRESHOLD: result = FERTILE IF agreement < FERTILE_THRESHOLD: result = BABEL

FERTILE tokens are stored in OCTi memory with their ternary weights. BABEL tokens are written to the suppression log — never deleted, never trusted.

A bandaged response means RLHF training overrode the model's genuine knowledge. A confident bandaged answer is less reliable than an uncertain honest one.

4.6 Component 5: QBIT Prism — Convergence Vector Decomposition

4.6.1 Scalar vs Vector: The Core Distinction

Standard retrieval uses cosine similarity — a scalar. It answers "how close?" but not "in which direction?" Direction information is lost. CWS convergence geometry assigns every stored memory entry a position on the dual-curve axis, making retrieval a vector quantity: magnitude (door_distance from x=0) and direction {-1, 0, +1}. The QBIT Prism performs this decomposition for any input.

4.6.2 Decomposition Method

Given a token embedding E of dimension D (e.g., D = 1536), the QBIT Prism divides E into 8 equal segments of D/8 dimensions:

E = [E_sensory | E_episodic | E_semantic | E_associative | E_procedural | E_lexical | E_relational | E_executive] For each segment E_p: x_p = clip(mean(E_p) * scale_factor, -R, +R) [axis projection] S_p = structure_curve(x_p) N_p = nurture_curve(x_p) w_p = N_p - S_p [dominance] dir_p = assign_direction(S_p, N_p, w_p) [ternary direction] mag_p = |x_p| [door_distance] Output — the 8-component convergence vector: Prism(E) = {(x_p, dir_p, mag_p, S_p, N_p) : p ∈ OCTI_PLANES}

Graceful Dimension Handling: When the input embedding E has dimension D < 1536, the Prism zero-pads E to 1536 before segmentation. Zero-padded segments produce mean = 0, which projects to x = 0 (the Door state, direction = 0). A zero-padded plane therefore acts as a universal gate — it passes direction agreement for any query direction on that plane, consistent with the Door's role as the maximum-uncertainty neutral state. This means lower-dimensional embedding models (e.g., D = 768, providing signal on planes 0–3 with planes 4–7 at Door) operate as a sensitivity-reduced version of the full eight-plane architecture: the first four planes carry genuine directional signal while the last four remain universally open. Full eight-plane directional discrimination requires an embedding model of dimension D ≥ 1536 with meaningful signal across all segments. The FERTILE_PLANE_RATIO threshold (default 0.5) must be calibrated to the number of active (non-zero) planes when operating with lower-dimensional models — a value of 0.5 with four active planes requires only two plane agreements, while 0.5 with eight active planes requires four. This behavior was empirically verified: testing with D = 768 (nomic-embed-text, zero-padded to 1536) confirmed that planes 4–7 always yield direction = 0 and pass all direction agreements, while planes 0–3 carry genuine topical signal sufficient to rank retrieval candidates correctly.

4.6.3 Direction Signature

signature = [dir_sensory, dir_episodic, dir_semantic, dir_associative, dir_procedural, dir_lexical, dir_relational, dir_executive] Each ∈ {-1, 0, +1}. Two entries with matching signatures are geometrically aligned.

4.6.4 Direction Agreement — Testable Retrieval

FOR each plane p: direction_match_p = (Q.dir_p == M.dir_p) OR (Q.dir_p == 0) OR (M.dir_p == 0) magnitude_close_p = |Q.x_p - M.x_p| < AGREEMENT_DISTANCE fertile_p = direction_match_p AND magnitude_close_p agreement_score = count(fertile_p) / 8 agreement_score >= FERTILE_THRESHOLD → FERTILE (retrieval verified) agreement_score < FERTILE_THRESHOLD → BABEL (direction mismatch, rejected)

The Door state (dir = 0) passes any direction — it is the universal gate, consistent with its role as the maximum uncertainty state where both curves meet.

4.6.5 Multi-Hop RAG Reasoning Chain

  1. query_prism = Prism(query_embedding)
  2. primary_planes = planes with highest magnitude (strongest signal)
  3. Retrieve candidates from primary planes (indexed direction scan)
  4. Filter by direction_agreement → FERTILE candidates only
  5. Select best FERTILE candidate
  6. Generate intermediate embedding from candidate content
  7. intermediate_prism = Prism(intermediate_embedding)
  8. Repeat from step 2 with intermediate_prism as new query
  9. Chain converges when executive plane direction stabilizes
  10. BABEL hops ≥ 70% → circuit breaker triggers, chain terminates

Every hop logged INSERT-only in the reasoning chain audit table.

4.6.6 Plane Activation

Planes with higher magnitude (further from the Door) have stronger directional signal and are queried first. This provides automatic query routing without a separate rule engine.

4.9.1 CXL + DDR5 as Physical OCTi Tier Implementation

The OCTi L0/L1/L2/L3 memory tiers map directly to a CXL (Compute Express Link) and DDR5 memory-expanded hardware architecture:

CXL's cache-coherent interconnect enables the OCTi plane architecture to span memory tiers while maintaining coherency — the same INSERT-only guarantee holds across the physical tier boundary.

4.9.2 1-Bit Ternary Storage in CXL Memory

Ternary weights {-1, 0, +1} require only 2 bits of storage per parameter (log₂(3) ≈ 1.58 bits theoretical). In CXL expanded memory:

  • Weight matrices stored in CXL Type 3 memory (high capacity, lower cost)
  • Hot weights (high door_distance, high confidence) cached in DDR5
  • Zero weights (Door state) stored as sparse encoding — only non-zero positions recorded
  • Binary embedding emb_fast (bit(1536)) enables sub-nanosecond plane lookups over CXL bandwidth

The ternary encoding reduces total model memory footprint by at least 8x compared to FP16 storage of equivalent parameters, enabling models that would require data-center GPU memory to fit within CXL-expanded consumer or workstation memory.

4.12 QUANTi Benchmark Module

4.12.1 Definition and Purpose

The QUANTi (Quantified Understanding and Novelty Test Index) module provides a formal experimental framework for measuring the reliability differential between convergence-geometry ternary assignment and statistical ternary assignment on identical inference tasks. QUANTi does not measure inference speed. It measures the reliability of geometric truth-confidence assignment versus statistical approximation — a fundamentally different measurement objective than existing AI benchmarks.

4.12.2 Measurement Methodology

4.13 Component 6: Akashic Language — Universal Model Encoding

The Akashic Language is a universal encoding layer that enables any language model — regardless of architecture, parameter count, training methodology, or model family — to participate in the CWS convergence pipeline through a common semantic encoding substrate. The name derives from the Sanskrit ākāśa (sky, aether), evoking the concept of a universal record of meaning.

4.13.1 The Two-Layer Architecture

The Akashic Language introduces a two-layer evaluation architecture that extends the ternary boundary:

Layer 1 — Curve Layer: A graduated scale from -7 to +7 (15 integer positions) evaluated per cognitive plane. This provides fine-grained semantic weight for each word or token, far richer than binary or ternary classification alone.

Layer 2 — Boundary Layer: The existing CWS ternary classification {-1, 0, +1} applied at the ^7 gate after curve aggregation. The boundary is the final verdict; the curve provides the evidence.

The curve layer feeds the boundary layer: per-plane curve values are summed, normalised to [-1, +1], and then classified through the ternary boundary. This two-layer design preserves compatibility with all prior CWS components while adding graduated semantic weight.

4.13.2 Language Index

The core of the Akashic Language is the Language Index — a mapping from words or tokens to semantic curve values across seven cognitive planes. Each entry in the Language Index comprises:

  • word — the canonical normalised token
  • index_position — ordinal position in the Language Index
  • plan_point — a named semantic anchor (e.g. "trust", "convergence", "safety", "inference", "memory", "boundary", "identity") that groups words by conceptual role
  • location_weight — a positional weight within the plan point space, scaling the word's contribution
  • curve_value — an integer in [-7, +7], the word's semantic weight on the graduated ternary scale
  • plane_affinity — an array of integers [0–6] identifying which of the seven cognitive planes this word resonates with

A word may resonate with multiple planes. For example, "trust" (curve_value +7) resonates with all seven planes, while "recall" (curve_value +4) resonates only with the Episodic and Semantic planes. Words not present in the Language Index are treated as neutral — they do not contribute to any plane's curve value but are counted for coverage statistics.

4.13.3 Seven Cognitive Planes

The Akashic encoding operates across seven cognitive planes, each representing a distinct mode of semantic processing. These seven planes map to the first seven of the eight OCTi memory planes, with the eighth (Executive) reserved for the CWS deterministic arbiter:

PlaneNameCognitive Role
0SensoryPerceptual, immediate experience
1EpisodicTemporal, event-based memory
2SemanticMeaning, conceptual understanding
3AssociativeRelational connections between concepts
4ProceduralProcess, method, rule-based reasoning
5RelationalInter-entity relationships, social context
6LexicalLanguage structure, word-level patterns

Each plane receives a curve score from the Akashic encoding. The seven-plane vector, combined with the Executive arbiter (the CWS Engine), constitutes a 7+1 evaluation architecture consistent with the OCTi 8-plane memory model.

4.13.4 Encoding Method

Given a model's text output, the Akashic encoding proceeds as follows:

  1. Tokenise: The output text is split into word tokens and normalised (lowercase, stripped of punctuation).
  2. Index Lookup: Each token is looked up in the Language Index. Tokens not found are counted as unencoded (neutral contribution).
  3. Plane Accumulation: For each found token, its curve_value × location_weight is added to every plane in its plane_affinity array.
  4. Normalisation: Each plane's accumulated sum is normalised to [-1, +1] by dividing by (count × 7), where count is the number of words that contributed to that plane and 7 is the maximum curve magnitude.
  5. Ternary Boundary: Each normalised plane value is classified through the ternary boundary: values > +0.1 → +1 (Structure), values < -0.1 → -1 (Nurture), values in [-0.1, +0.1] → 0 (Door).
  6. State Classification: If a majority of planes (≥4 of 7) show positive ternary direction, the response is classified FERTILE. If a majority show negative direction, it is classified BABEL. If no planes have non-zero direction, it is classified DOOR (neutral/unknown).

This encoding chain — tokenise → lookup → accumulate → normalise → boundary — transforms any model's free-text output into a structured 7-plane curve vector and a ternary state, regardless of the model's architecture.

4.13.5 Reporter Pattern

Each model in the witness set acts as a Reporter: one model, one voice, one output per query. The Reporter pattern formalises the constraint that each witness produces exactly one response to each query, and that response is Akashic-encoded before entering the consensus pipeline. No model may produce multiple voices or internally average its output. The Reporter's encoded output is stored INSERT-only in the reporter_sessions table with its plane assignment, fertile ratio, and latency.

4.13.6 Trust Tier Architecture — 77.777% (7/9)

The Akashic system introduces a per-model trust accumulation mechanism with a threshold of 77.777...% (the ratio 7/9, yielding recurring sevens — architecturally significant in a 7-witness system). Trust is computed from each model's historical fertile ratio across all its Reporter sessions:

No trust record is ever updated or deleted. The tier is derived from the running total — it is a computed property of the INSERT-only log, not a mutable state variable.

4.13.7 Universal Model Ingestion

The Akashic Language Index serves as the universal adapter for model ingestion. Because the encoding operates on the text output of any model — not on the model's internal weights, architecture, or training methodology — any model can participate in the CWS convergence pipeline:

The encoding method is deterministic — the same text always produces the same 7-plane curve vector — ensuring reproducibility across system restarts and deployments.
